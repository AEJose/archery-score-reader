# 記分紙方向自動處理策略（2026-05）

本文件說明目前偵測 pipeline 如何處理以下真實情境：

- 影像可能 **上下顛倒（180°）**
- 影像可能是 **直式拍攝（portrait）** 而非橫式
- 使用者可能手持拍照，方向不固定（0° / 90° / 180° / 270°）

---

## 1. 核心策略：多方向嘗試 + 最佳候選挑選

在 `SheetRenderer._detect_cells()` 中，現在會對同一張灰階影像嘗試 4 種旋轉角度：

1. `0°`（原圖）
2. `90°`
3. `180°`
4. `270°`

每一個方向都會完整跑一次「區域偵測 + 區內格線偵測 + cell 切分」。
最後用「偵測到的 cell 總數」作為分數，挑選分數最高的方向作為最終結果。

> 這個策略可以自然涵蓋 portrait 與 upside-down 影像，不需要先做方向分類器。

---

## 2. 每個方向的偵測流程

每個旋轉方向會走 `_detect_cells_for_orientation()`：

1. `_detect_target_regions()`：偵測候選記分區（濾掉不合理框）
2. `_detect_region_grid_lines()`：在每個記分區內做自適應二值化與形態學線條抽取
3. 依相鄰水平/垂直線建立 cell，並做行高、欄寬過濾

這個流程是「區域內偵測」，不像舊版是全圖掃描，因此對傾斜/曝光變化更穩定。

---

## 3. 座標回映射（rotated → original）

因為最終繪字要落在原始影像座標，當最佳方向不是 0° 時，會把每個 cell 的四角座標映射回原圖。

目前支援的映射：

- `rot_k=1`（90°）：`(x, y) -> (y, H-1-x)`
- `rot_k=2`（180°）：`(x, y) -> (W-1-x, H-1-y)`
- `rot_k=3`（270°）：`(x, y) -> (W-1-y, x)`

再以映射後四角重建 axis-aligned bounding box，確保後續文字繪製正常。

---

## 4. 為什麼這能處理你提到的兩個問題

### A) 上下顛倒（180°）

- 180° 分支會直接把內容轉正來偵測。
- 若該分支得到最多有效 cell，就會被選為最佳方向。
- 回映射後座標回到原圖系統，繪製位置仍正確。

### B) 直式照片（portrait）

- 90° 或 270° 分支通常會得到更高的格線規律與更多有效 cell。
- 分數機制會自動選到該分支，不需手動指定方向。

---

## 5. 已知限制與下一步

1. 目前以「cell 數量」當單一分數，極端噪聲下仍可能誤選方向。
2. 尚未加入 OCR 信心分數做二次排序（未來可用）。
3. 若影像存在大角度透視扭曲，仍建議先加 deskew/perspective correction。

建議下一步：

- 加入複合分數（cell 數、行列規律、欄寬方差、OCR confidence）
- 對真實資料集紀錄各方向分數，建立方向選擇的可觀測性
